## 1. 产品概述与灵魂

### 1.1 产品基本信息

| 项目 | 内容 |
|------|------|
| 产品名称 | Druk（酒精计划） |
| 平台 | iOS 优先，SwiftUI 原生开发 |
| 目标用户 | 25-40 岁，有品质意识的饮酒者 |
| 商业模式 | 免费 + 一次性买断高级功能（无订阅） |
| 数据存储 | 全本地，无账号，无云同步 |

### 1.2 产品灵魂

本产品取材自 2020 年丹麦电影《酒精计划》（Druk，导演托马斯·温特伯格，主演麦斯·米科尔森）。

电影核心命题来自挪威哲学家芬·斯卡德鲁德（Finn Skårderud）：人类血液中天生缺少 0.05% 的酒精，补足这个量才能达到最理想的生命状态——更放松、更有创造力、更敢爱。四位中年教师以此为出发点做了一场生活实验，最终每个人都得到了不同的答案。

**Druk app 的产品立场：**

- 不是劝人喝酒的工具
- 不是说教用户的健康应用
- 是一面镜子——帮你看清自己此刻在哪里，让你自己决定该在哪里停下来

### 1.3 核心流程

```
打开 app → 输入个人信息（体重/性别）→ 选择国家标准
→ 记录饮酒（酒类 + 度数 + 饮量）→ 查看实时 BAC
→ 查看清醒曲线与倒计时 → 保存记录（可选）
```

## 3. 核心功能模块

### 3.1 BAC 计算器

#### 3.1.1 计算公式

采用国际法医毒理学标准 Widmark 公式（1932年，Erik Widmark 建立，至今仍是行业标准）：

```
BAC(t) = [A / (W × r)] × 100 - (β × t)

A  = 酒精摄入总克数
     = 饮量(ml) × ABV(%) × 0.789（酒精密度）
W  = 体重（g）
r  = Widmark 分布系数
     男性：0.68，女性：0.55
β  = 酒精消除速率（见 3.2 节）
t  = 饮酒开始后经过的小时数
```

**标准饮品换算：**

| 酒类 | 默认 ABV | 标准单份 |
|------|---------|---------|
| 啤酒 | 5% | 350 ml（一瓶） |
| 葡萄酒 | 12% | 150 ml（一杯） |
| 香槟 | 11% | 120 ml |
| 威士忌/白兰地 | 40% | 30 ml（一shot） |
| 中国白酒 | 52% | 50 ml（一两） |
| 清酒/黄酒 | 15% | 100 ml |

#### 3.1.2 输入控件

- **酒精度数**：滑块，范围 0.01% - 70%，精度 0.1%，实时更新 BAC
- **饮酒量**：数字输入框 + 单位切换（ml / oz / 两），三种单位实时换算
- **饮酒时长**：记录开始喝酒的时间戳，用于计算时间轴中的当前 BAC
- **快捷预设芯片**（水平滚动）：点击自动填充酒类对应的默认 ABV
  - 🍺 啤酒 · 🍷 葡萄酒 · 🥂 香槟 · 🥃 威士忌 · 🍶 白酒 · 🍵 清酒
- **目标状态芯片**：反算需要摄入多少酒精可达到该 BAC
  - 微醺 0.05% · 社交 0.08% · 清醒（触发倒计时模式）

#### 3.1.3 BAC 状态显示

主界面大字体实时显示当前 BAC（保留 3 位小数，如 `0.052%`），配合背景氛围颜色和状态标签：

| BAC 范围 | 状态名 | 背景色调 |
|---------|--------|---------|
| 0.000% | 清醒如水 | 深棕中性 |
| 0.001% - 0.019% | 微微感觉 | 深棕偏暖 |
| 0.020% - 0.049% | 恰到好处 | 琥珀暖调 |
| 0.050% | 哲学黄金点 | 琥珀金高亮 |
| 0.051% - 0.079% | 微微漂浮 | 橙琥珀 |
| 0.080% - 0.099% | 超过警戒线 | 橙红 |
| ≥ 0.100% | 请停下来 | 深红 |

### 3.2 清醒预测曲线与倒计时

> 本功能为次要功能，在主界面以可展开卡片形式呈现，不占主界面核心位置。

#### 3.2.1 功能定位

将 BAC 随时间变化的完整过程用曲线可视化，结合倒计时，让用户直观理解「自己在哪里，还有多久」。

#### 3.2.2 曲线模型

曲线分为两个阶段：

**阶段一：吸收上升期（约 45 分钟）**

```
t ∈ [0, 0.75h]：BAC(t) = peak_BAC × (t / 0.75)
```

酒精从胃和小肠吸收入血，BAC 线性上升至峰值。空腹时峰值出现更早（约 30 分钟），餐后更晚（约 60-90 分钟）。

**阶段二：代谢消除期**

```
t > 0.75h：BAC(t) = max(0, peak_BAC - β × (t - 0.75))
```

肝脏持续氧化代谢，BAC 以近似线性速率下降至 0。

#### 3.2.3 消除速率 β 的科学依据与展示策略

**学术背景（必须在 app 内诚实披露）：**

根据 PubMed 循证综述（Jones, 2010）及多项法医毒理学研究，酒精消除速率的生理范围是 0.010 - 0.035 g/100mL/h，个体差异可达 3 倍以上。在法庭上，专家通常采用 0.010%、0.015%、0.020% 三档进行区间估算而非单一值。

| 代谢档位 | β 值 | 适用人群描述 |
|---------|------|------------|
| 慢（保守估算）| 0.012%/h | 空腹饮酒、女性、体重较轻、肝功能一般 |
| 中（标准值）| 0.015%/h | 餐后饮酒的一般成年人，法医毒理学默认值 |
| 快（乐观估算）| 0.020%/h | 餐后、男性、体重较大、代谢旺盛 |

**关键约束：**
- 咖啡、冷水澡、运动、大量饮水均无法加速酒精代谢
- 整体误差范围 ±20%
- 模型在吸收期和极低 BAC 阶段（< 0.02 g/L）精度下降

**UI 展示策略：不给出单一精确时间，给出区间**

错误示范：「你将在 4 小时 23 分后完全清醒」

正确示范：「预计清醒时间：4-6 小时后」（基于慢/快两档计算上下限）

#### 3.2.4 曲线 UI 需求

**曲线视觉设计：**

- 已过时间段：实线（琥珀色），代表确定的历史
- 未来预测段：虚线（半透明琥珀），代表估算
- 不确定区间带：慢/快两档之间的淡色填充区域，直觉传达「这是估算范围」
- 当前位置：实心圆点标记在曲线上，显示实时 BAC 数值
- 「NOW」竖向虚线：当前时刻的垂直参考线

**参考线（水平虚线）：**

| 参考线 | 值 | 颜色 | 说明 |
|--------|-----|------|------|
| 中国醉驾 | 0.08% | 红色 | 超过即刑事犯罪 |
| 中国饮酒驾车 | 0.02% | 橙色 | 超过即违法 |
| 哲学黄金点 | 0.05% | 金色（虚线） | 斯卡德鲁德理论值 |

**色带区域（背景填充）：**

- 红色区（> 0.08%）：危险区
- 橙色区（0.02% - 0.08%）：法律警告区
- 绿色区（0% - 0.02%）：安全区

#### 3.2.5 倒计时显示逻辑

倒计时目标根据当前 BAC 状态动态切换，优先显示最紧迫的用户关切：

| 当前 BAC | 倒计时目标 | 显示文案示例 |
|---------|-----------|------------|
| ≥ 0.08% | 降至 0.08% 以下 | 距离醉驾线以下：2h 30m |
| 0.02% - 0.08% | 降至 0.02% 以下 | 距离中国法定安全：3h 10m |
| 0.001% - 0.02% | 降至 0.00% | 距离完全清醒：45m |
| < 0.001% | 已清醒 | 酒精已代谢完毕 |

#### 3.2.6 「我要 X 点开车」反向计算功能

用户输入目标时间（如「明早 9:00」），系统反算：

- 当前能继续喝多少（折算成标准杯数）
- 还需要等待多久才可以达到法定安全线
- 以红/黄/绿色标注三种状态：超标 / 接近线 / 安全

### 3.5 饮酒记录系统

#### 3.5.1 设计原则

记录功能不是「数据警察」，而是「生活见证者」。

- 记录界面的文案基调：「今晚，发生了什么？」而非「输入你的饮酒量」
- 年度报告不叫「饮酒分析」，叫「你的那些夜晚」
- 每次保存记录后，随机显示一句《酒精计划》相关台词
- 数据展示不做道德评判，只做客观呈现

#### 3.5.2 真实用户需求分层

**场景一：聚会实时记录**
需求：不超标、保持最佳状态
设计：主界面「+1 杯」快捷按钮，点击后选择酒类，BAC 实时更新

**场景二：事后复盘**
需求：理解宿醉原因，避免再次发生
设计：支持次日补录，饮酒开始时间可手动回调，宿醉评分可次日填写

**场景三：健康管理**
需求：量化饮酒习惯，自我约束
设计：周/月纯酒精摄入克数统计，叠加 WHO 安全参考线（男 ≤140g/周，女 ≤70g/周）

**场景四：记录美好**
需求：留存某个特别的夜晚，不只是数据
设计：支持添加地点、酒款备注、心情标签、一张照片

**场景五：驾车规划**
需求：喝之前就想好回程
设计：「我要 X 点开车」功能入口，反向计算可饮量

#### 3.5.3 单条记录的信息结构

**基础层（必填或系统自动）：**

| 字段 | 说明 |
|------|------|
| 时间戳 | 开始喝酒的时间（支持手动修改） |
| 酒品清单 | 支持多条：酒类 + ABV% + 饮量（ml） |
| 计算值 | 系统自动计算：总纯酒精克数、峰值 BAC |

**体验层（选填）：**

| 字段 | 说明 | 示例 |
|------|------|------|
| 场合标签 | 单选 | 工作应酬 / 朋友聚会 / 独酌 / 约会 / 纪念日 / 其他 |
| 地点 | 手动输入或地图选点 | 「外滩某酒吧」 |
| 心情 | 5 种情绪 emoji 选择 | 😌 平静 / 😄 开心 / 🥹 感动 / 😔 低落 / 🥳 狂欢 |
| 酒款备注 | 140 字文字输入 | 「第一次喝到这款限定，值得记念」 |
| 照片 | 最多 1 张 | 酒标、现场照片等 |

**反思层（事后推送触发）：**

| 字段 | 触发时机 | 说明 |
|------|---------|------|
| 宿醉评分 | 次日早上推送 | 0-5 分，「第二天的感受如何？」 |
| 简短感想 | 可选，随时填写 | 140 字，类朋友圈风格 |

#### 3.5.4 历史记录视图

**列表视图（默认）：**

每条记录卡片展示：日期时间、场合标签、酒品摘要、峰值 BAC 徽章、宿醉评分（若已填写）

**日历热图视图：**

全年日历，每天用色块深浅表示当日饮酒量（空白 / 浅 / 中 / 深），类 GitHub contribution graph

**趋势图视图：**

7 天 / 30 天 / 12 个月可切换。柱状图显示每次最高 BAC 或纯酒精克数，叠加 WHO 安全参考线（虚线，不做强制提醒，用户自行感受）

**年度报告（「你的那些夜晚」）：**

每年 1 月 1 日解锁，展示：
- 全年饮酒次数

## 4. 内容系统：BAC 状态提示词完整表格

### 4.1 设计定位

这是 Druk 区别于所有 BAC 计算器的核心差异化。

每个 BAC 状态节点对应一条**主提示词**（大字展示，随状态切换）和若干**轮播备选句**（同一状态内每 90 秒随机切换一条）。提示词分三种类型：

- **哲思型**：来自电影台词、哲学家引言、生活感悟，营造深夜广播节目质感
- **科学型**：基于研究数据，用诗意语言描述当前 BAC 对身体/心理的真实影响
- **提示型**：在法律临界点和危险区域，以非说教的方式传递安全信息

所有提示词的语气原则：**像一个见过世面的老朋友在跟你说话，而不是卫生部在发公告。**

### 4.4 0.025% 微醺助性点的科学背景（PRD 内部注释）

此节为产品内容设计的学术依据，不直接展示给用户，供文案撰写参考。

**核心研究结论：**

1977 年发表于 PubMed 的研究对 16 名青年男性进行了测试，结论是：最低酒精剂量（BAC 约 0.025%）时生理性唤起（以阴茎直径变化衡量）达到峰值；而 BAC 0.05% 及以上时，生理性唤起出现显著抑制。

1980 年代的多项研究表明，BAC 约 0.05% 开始，酒精对男性生理性唤起产生抑制效果；更低剂量时，酒精对唤起无影响或有轻微促进作用。

酒精会提升睾酮和雌二醇水平，由于睾酮部分影响女性性欲强度，这在生理机制上解释了为何低度饮酒可能增加性兴趣。同时研究表明，主观性唤起的上升（感觉更aroused）与生理性唤起的实际变化之间存在显著落差——心理感受先行，生理响应其实在下降。

低至中度饮酒（通常定义为女性 1-2 标准杯、男性 2-3 标准杯）可产生欣快感和焦虑降低，从而暂时提升对性活动的兴趣。然而随着饮酒量增加，酒精对中枢神经系统的抑制作用更加明显，导致性表现下降、高潮困难甚至勃起障碍。

**文案使用指南：**
- 0.025% 是感官敏感性的生理峰值，可以用「感官苏醒」「触碰变得真实」等意象
- 这个点是性暗示最合适的位置——既有科学依据，又不至于过度
- 0.050% 之后要在哲学浪漫的同时加入「但身体是诚实的」这类温柔的反转
- 0.080% 以上完全切换为安全提示主导，不再出现性相关暗示

### 5.5 技术实现说明

- 使用 UIGraphicsImageRenderer 将 SwiftUI 视图渲染为 UIImage
- 卡片内容在离屏 View 中按固定尺寸渲染，不依赖屏幕分辨率
- 台词、BAC 数值、状态名等动态内容在渲染前锁定快照，分享后不随 app 状态变化
- 分享功能不上传任何数据至服务器，图片仅在设备本地生成

### 5.6 不包含的内容（隐私边界）

以下内容不出现在任何分享卡片中：

- 体重、性别等生物特征数据
- 精确地理位置坐标（仅显示城市级手动输入文字）
- 宿醉评分
- 历史记录的完整列表

## 6. 小组件（Widget）需求

### 5.1 定位

清醒倒计时为次要功能，不在主界面占主要位置，以独立 Widget 形式发挥最大价值。

### 5.2 Widget 规格

| 尺寸 | 内容 |
|------|------|
| 小组件（2×2）| 倒计时大字（如「4h 20m」）+ 状态色点（红/橙/绿） |
| 中组件（4×2）| 倒计时 + 当前 BAC 数值 + 目标说明 + 简短进度条 |
| 锁屏组件 | 倒计时数字，深色背景，白色文字 |
| 灵动岛（Live Activity）| 实时 BAC + 倒计时，类外卖进度条形式 |

### 5.3 更新策略

- 使用 `WidgetKit` Timeline，每 15 分钟刷新一次（BAC 变化频率）
- 灵动岛使用 `ActivityKit`，实时更新
- 当 BAC 降至设定阈值（如 0.08%、0.02%）时，发送本地通知提醒
- Widget 底部常显免责声明（11px）：「估算值，误差 ±20%，请勿用于判断驾车安全」

### 5.4 Widget 免责声明展示要求

首次添加 Widget 时弹出一次性确认：「此倒计时为科学估算，个体代谢差异显著，实际清醒时间可能提前或推迟 1-2 小时。请勿以此判断驾车安全。」确认后不再重复显示。

## 7. UI 设计规范

### 6.1 视觉定位

从参考 UI（GO Club 健步 app）提取的通用语言：暗色背景 + 高饱和强调色 + 纯白卡片浮层 + 超重数字。迁移到 Druk 时，将色调从「白天运动感」调整为「深夜酒馆感」。

### 6.2 色彩系统

| 元素 | 色值 | 用途 |
|------|------|------|
| 主背景 | `#0D0A08`（近黑深棕）| 页面底色，如吧台木材质感 |
| 强调色 · 金 | `#C8862A`（琥珀金）| 主要按钮、大数字、选中态、曲线实线 |
| 强调色 · 红 | `#8B1A2B`（深酒红）| 危险状态 BAC、超标警告 |
| 卡片背景 | `#1A1410`（深棕）| 卡片、输入区背景 |
| 文字 · 主 | `#F0E6D3`（奶油白）| 主要文字、数字 |
| 文字 · 次 | `#8C7B6B`（暖灰）| 辅助信息、标签、说明文字 |
| 安全绿 | `#2A6B4A` | 安全状态、清醒倒计时完成 |
| 法律橙 | `#BA7517` | 法律警告区、0.02%-0.08% |

### 6.3 字体规范

| 用途 | 字体 | 规格 | 颜色 |
|------|------|------|------|
| BAC 主数字 | SF Mono（等宽）| 72pt | 琥珀金 `#C8862A` |
| 倒计时数字 | SF Mono | 48pt | 奶油白 |
| 状态标签 | Georgia 衬线体 | 16pt | 奶油白 |
| 台词引用 | Georgia Italic | 16pt | 奶油白，左对齐，带 `"` 装饰 |
| UI 标签/按钮 | SF Pro Rounded | 14pt | 暖灰 |
| 法律/免责小字 | SF Pro | 11pt | 暖灰 60% 透明度 |

### 6.4 关键动效设计

| 动效 | 描述 |
|------|------|
| BAC 数字变化 | 机械计数器式滚动，逐位更新 |
| 状态切换背景 | 颜色缓慢渐变，3 秒过渡，不闪烁 |
| 曲线绘制 | 进入曲线页面时，曲线从左向右「生长」动画 |
| 当前点移动 | 实时 BAC 圆点沿曲线移动，阻尼弹簧动效 |
| 台词出现 | 从底部渐显，停留 8 秒后渐隐 |
| 保存记录 | 底部保存按钮按下后，卡片「飞入」历史记录动画 |
| 完全清醒 | 背景渐变回深棕中性色，配合提示文字 |

### 6.5 布局原则

- 卡片式布局，玻璃拟态效果，垂直滚动
- 移动端优先，适配 iPhone 所有屏幕尺寸（iPhone 13 mini 至 iPhone 16 Pro Max）
- 触控优化：最小点击区域 44×44pt
- 横屏适配：确保内容展示正常，曲线图切为宽屏比例
- Dark Mode 专属设计（此 app 没有浅色模式，始终为深色）

## 8. 非功能需求

### 7.1 隐私与数据

- 所有用户数据仅存储在本地设备（Core Data + Keychain）
- 无账号注册，无云同步，无任何数据上传
- 无广告，无追踪，无第三方 SDK（除 Apple 原生框架外）
- 首次启动一次性法律声明（用户需主动确认）：
  「本 app 基于 Widmark 公式估算 BAC，误差 ±20%，仅供教育与参考用途，不构成驾车安全建议。饮酒后请勿驾车。数据仅存储在您的设备本地。」

### 7.2 性能

- BAC 实时计算响应时间 < 16ms（不阻塞主线程）
- 曲线渲染帧率 ≥ 60fps
- 冷启动时间 < 1 秒
- Widget 刷新延迟 < 15 分钟

### 7.3 无障碍

- 支持动态字体（Dynamic Type），最大字号下布局不破损
- VoiceOver 完整适配（BAC 数值、状态、倒计时均有语义描述）
- 减弱动效模式（Reduce Motion）下，所有动效退化为简单淡入淡出

### 7.4 法律

- app 内所有 BAC 相关功能页面底部常显：「仅供参考，不可用于判断驾车安全」
- 法规数据标注来源与生效时间，提示用户以当地实际法规为准
- 年龄提示：首次启动提示本 app 面向合法饮酒年龄用户（中国：18 岁以上）
